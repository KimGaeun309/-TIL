# URL의 이해

같은 파일이지만 URL 값을 다르게 함으로써 PHP 애플리케이션이 클라이언트(웹 브라우저)마다 다른 웹 페이지를 보여줄 수 있다.

## URL의 형식

> `http://opentutorials.org:3000/main?id=HTML&page=12`
>
> 와 같은 URL이 있을 때,
>
> - `http` 는 protocol, 통신 규칙이다. 즉 사용자가 서버에 접속할 때 어떤 방식으로 통신할 것인가에 대한 부분이다. 웹 서버와 브라우저가 정보를 주고받기 위한 것.
> - `opentutorials.org` 는 domain name 혹은 host이다. host는 인터텟에 접속되어있는 각각의 컴퓨터를 말한다. 즉, 이는 어떤 특정한 인터넷에 연결되어 있는 컴퓨터를 가리키는 주소이다.
> - `3000` 는 port 번호이다. 한 대의 컴퓨터 안에 여러 개의 서버가 있을 수 있어 클라이언트가 그 중 어느 서버와 접속할지 알기 위해 사용한다.
> - `main` 은 path이다. 그 컴퓨터 안의 어느 디렉토리의 어느 파일인지를 가리킨다.
> - `id=HTML&page=12` 는 query string이다. 이 값을 변경하면 이 앞에 있는 웹 서버에게 어떤 데이터를 전달할 수 있다. 내가 읽고 싶은 정보는 HTML이고 12page임을 나타내고 있다. query string의 시작은 ? 로 하기로 약속되어 있으며 값과 값은 & 로 구분하며 값의 이름과 값은 = 로 구분하기로 약속되어 있다.

즉, query string 을 통해 사용자마다 다른 페이지를 보여줄 수 있다.
